Line One 网页话机产品介绍

产品定位

Line One 是一套本地部署的 WebRTC 到 VOS3000 SIP/UDP 网页话机网关。它让用户可以在浏览器里完成 SIP 注册、拨号、通话、挂机、音频文件播报和信令查看，不需要安装传统软电话客户端。

系统运行在本地电脑或服务器上，网页侧使用 WebRTC，后端使用 Go 网关对接 VOS3000 的 SIP/UDP 与 RTP 媒体流。

Browser WebRTC <-> Go/Pion Gateway <-> SIP/UDP + RTP <-> VOS3000

核心能力

  • 网页 SIP 注册：支持在页面填写注册地址、账号和密码，不需要修改配置文件。
  • 多话机配置：浏览器本地可保存多个 SIP 账号，进入页面后选择要启用的话机。
  • UDP 对接 VOS3000：面向 VOS3000 话机注册与外呼场景，使用 SIP/UDP。
  • 浏览器拨号：支持号码输入、拨号键盘、通话计时、主动挂机。
  • 最近通话：保留最近 10 通呼出记录，点击历史号码只回填拨号框，避免误触外呼。
  • 信令交互视图：按主叫、被叫分栏展示 SIP 交互过程，并可查看完整原始报文。
  • 历史信令查看：历史通话可按 Call-ID 查看当次信令过程。
  • 音频模式选择：支持耳麦实时通话和音频文件播报。
  • 播报后自动挂机：选择文件播报时，电话接通后服务端直接向被叫发送 RTP 音频，播放结束后自动发送 BYE。
  • 编码选择：支持 G.711U/PCMU 与 G.711A/PCMA。G.729 已预留展示，但需要外部转码组件。

典型使用场景

本地网页话机

适用于只能在指定本地网络或固定出口 IP 注册 VOS3000 的场景。服务运行在允许注册的本机或内网服务器上，用户通过浏览器操作话机。

客服与测试外呼

支持快速注册、拨号、查看信令和通话结果。信令面板可帮助定位 VOS3000 返回码、媒体地址、ACK/BYE 是否正常等问题。

自动语音播报

在音频文件模式下，用户选择音频文件后发起呼叫。被叫接通后，系统直接播放文件音频，不使用耳麦，不需要人工讲话。播放结束后系统自动挂机。

SIP/RTP 调试

实时展示 INVITE、100 Trying、183、180、200 OK、ACK、BYE 等信令，并记录 RTP 上行、下行统计，方便排查注册、呼叫建立和媒体单通问题。

页面功能概览

话机配置

首次进入系统时，如果没有已保存配置，会显示新增 SIP 话机表单。注册成功后，账号配置保存在当前浏览器本地存储中。后续进入页面时，可以从配置列表中选择要注册的话机。

配置包含 SIP 注册地址、SIP 账号和 SIP 密码。

拨号与通话

拨号页提供号码输入、拨号键盘、编码选择和音频来源选择。呼叫期间会锁定关键配置，避免通话中切换账号或修改音频导致会话异常。

最近通话

系统保存最近 10 通呼出记录。点击历史号码只会填入拨号框，需要用户再次点击“呼叫”才会真正外呼，降低误触风险。

信令交互

信令交互窗口参考 VOS3000 的查看方式，将主叫发送和被叫返回分栏展示。列表中默认展示简短信令名称，例如 INVITE、100 Trying、200 OK、ACK、BYE。点击任意一条可在右侧查看完整 SIP 报文。

技术架构

前端

  • 原生 HTML/CSS/JavaScript。
  • WebRTC 获取麦克风音轨。
  • 本地存储 SIP 配置和最近通话记录。
  • 实时轮询网关状态和信令记录。

后端

  • Go HTTP 服务。
  • Pion WebRTC 处理浏览器媒体。
  • 自研轻量 SIP/UDP 客户端。
  • Digest 认证注册。
  • RTP 转发与 G.711 编码。
  • 内存信令记录与 Call-ID 过滤。

媒体链路

耳麦通话模式：

Microphone -> Browser WebRTC -> Gateway -> RTP/G.711 -> VOS3000

音频文件播报模式：

Audio File -> Gateway PCM/G.711 -> RTP -> VOS3000 -> 被叫

文件播报模式不依赖浏览器麦克风，适合固定语音通知、提示音或测试音频。

网络与部署要求

  • VOS3000 需要允许 SIP/UDP 注册。
  • VOS3000 需要支持 G.711U 或 G.711A。
  • 本机或服务器的 SIP/RTP 端口需要能被 VOS3000 访问。
  • 如果部署在 NAT 后，需要端口映射，或依赖 VOS3000 的对称 SIP/RTP 能力。
  • 非 localhost 访问网页时，浏览器麦克风权限通常要求 HTTPS。

默认端口：HTTP 127.0.0.1:8080，SIP 本地 UDP 5066，RTP UDP 40000。

启动方式

安装 Go 1.22 或更新版本后，在项目目录执行：

go mod tidy
go run ./cmd/gateway

然后打开 http://127.0.0.1:8080。

命令行模式

系统也提供 CLI 模式，可用于服务器测试或自动播报：

$env:SIP_PASSWORD="example-password"
go run ./cmd/phone-cli `
  -server 127.0.0.1:5060 `
  -id 1000 `
  -call 1001 `
  -codec pcmu `
  -audio .\prompt.wav

当前限制

  • 当前版本为单账号、单路通话。
  • G.729 需要额外转码组件，当前不直接支持。
  • 信令和通话记录主要保存在内存和浏览器本地，未接入数据库。
  • 生产环境暴露网页访问时，需要增加登录鉴权、HTTPS、访问控制和审计能力。

后续规划

  • 支持多路并发通话和 RTP 端口池。
  • 增加数据库保存通话记录和信令记录。
  • 增加用户登录、权限控制和配置加密。
  • 支持 Prometheus 指标、结构化日志和告警。
  • 评估合法合规的 G.729 转码方案。
  • 提供 CentOS / systemd / Docker 的标准化部署包。

总结

Line One 的目标是提供一个轻量、可控、面向 VOS3000 的本地网页话机。它兼顾日常拨号、自动语音播报和 SIP/RTP 调试能力，适合本地注册受限、需要快速排查信令媒体问题、或希望用网页替代传统软电话的场景。
